# 작업서 — 샤오장의 중국어학습

VSCode에서 바이브코딩으로 이어서 작업할 수 있도록 정리한 인수인계 문서입니다.
이 문서 + `index.html` 하나만 있으면 어떤 AI 코딩 툴(Claude Code, Cursor, Copilot 등)에게 그대로 붙여넣고 작업을 이어갈 수 있어요.

> 이 문서는 2026-07-03에 처음부터 새로 만든 버전을 기준으로 작성되었습니다. Claude 아티팩트 환경(`window.storage`, 인증 없는 Claude API 직접 호출)에 의존하던 이전 설계는 배제하고, 처음부터 독립 배포(GitHub Pages 등) 가능한 형태로 구현했습니다.

## 1. 프로젝트 개요

- **이름**: 샤오장의 중국어학습
- **형태**: 빌드 도구 없는 단일 HTML 파일 (Vanilla JS, 프레임워크 없음)
- **목적**: 개인용 중국어 단어 암기 + 문장 작문 연습 웹앱
- **데이터**: 391개 단어(HSK1 111 · HSK2 93 · HSKK초급 25 · 회화 30 · 업무 30 · 금융 23 · 거래소 23 · IT/개발 30 · 암호화폐 26). 앱 내 `단어장 관리` 탭에서 자유롭게 추가/수정/삭제 가능합니다.
- **파일**: `index.html` 1개 (HTML+CSS+JS 전부 인라인)

## 2. 기술 스택 & 선택 이유

| 항목 | 선택 | 이유 |
|---|---|---|

| 저장소 (예정) | Supabase (Postgres, 무료 티어) — TODO 1번 | 여러 기기·브라우저 동기화가 필요해지면 전환. 아직 프로젝트를 만들지 않았고 코드에도 반영 안 됨 |
| 배포 | GitHub Pages | 무료, 정적 파일이라 서버 불필요. `https://silllvereun.github.io/xiaozhang-chinese/`에 배포 완료 |

## 3. 알아둬야 할 제약사항

### 3-1. 문장 채점은 문법 검사가 아님
`gradeSentence(word, text)`는 4가지 형식적 조건만 확인합니다: (1) 목표 한자 단어 포함, (2) 한자 4자 이상, (3) 문장부호(。！？!?)로 끝맺음, (4) 한글 미포함. 어순이나 조사, 문법 오류는 잡아내지 못합니다. 더 정교한 첨삭이 필요하면 아래 옵션을 고려하세요.

- **옵션 A**: 본인 Anthropic API 키로 서버리스 프록시(Cloudflare Worker, Vercel Function 등)를 만들고, 그 프록시로 `fetch`해서 AI 첨삭을 추가 기능으로 얹기 (규칙 기반 채점은 항상 기본으로 유지 — progressive enhancement).
- 지금은 A를 구현하지 않은 상태입니다. TODO 2번 참고.

### 3-2. 저장소가 아직 localStorage
`단어장 관리`에서 만든 데이터는 그 브라우저/기기에만 남습니다. 다른 기기에서 이어보려면 "백업 내보내기"로 JSON을 뽑아 다른 기기에서 "백업 가져오기"로 불러오세요. 여러 기기 실시간 동기화가 필요해지면 TODO 1번(Supabase 마이그레이션) 진행.

## 5. 코드 아키텍처 요약

상태 관리는 전역 `state` 객체 하나 + `render()` 함수가 매번 `app.innerHTML`을 통짜로 다시 그리는 단순한 패턴입니다 (가상 DOM 없음). 검색 필터(`단어장 관리`)만 예외적으로 `renderManage()` 내부에서 DOM을 직접 필터링해 매 타이핑마다 전체 재렌더를 피합니다.

### 5-1. 데이터 계층
| 함수/변수 | 역할 |
|---|---|
| `RAW_WORDS` | 카테고리별 `한자\|병음\|뜻` 텍스트 블록 (seed 원본) |
| `buildInitialWords()` | `RAW_WORDS`를 파싱해 `INITIAL_WORDS` 배열 생성 |
| `WORDS` (let, 배열) | 실제 런타임에 쓰이는 단어 목록. 저장소에서 복원되거나 없으면 `INITIAL_WORDS`로 시드 |
| `wordStats` (let, 객체) | `{ [wordId]: {wrong, correct, seen, passed, lastSeen} }` — 단어별 학습 통계 |
| `dbGet/dbSet/dbDelete(key)` | 저장소 추상화. 현재는 `localStorage` 기반. **Supabase로 옮길 때 이 3개 함수 내부만 교체하면 됨** |
| `loadWordList()` / `persistWordList()` | `WORDS` 배열 저장소 동기화 (키: `wordList`, 300ms 디바운스) |
| `loadStats()` / `persistStats()` | `wordStats` 저장소 동기화 (키: `wordStats`, 300ms 디바운스) |
| `addWord / updateWord / deleteWord / toggleStar` | 단어장 CRUD. 항상 `persistWordList()` 동반 |

### 5-2. 출제 로직
| 함수 | 역할 |
|---|---|
| `pool()` | 현재 필터(`selectedGroups`, `starredOnly`, `reviewMode`) 기준으로 후보 단어 반환 |
| `weightedSample(arr, n)` | 오답 횟수(`wrongCountOf`) + 안 본 지 오래된 정도(`daysSinceSeen`)를 가중치로 반영한 비복원 랜덤 샘플링 |
| `normalize(s)` / `stripTones(s)` | 병음 정답 비교용 — 성조 기호(NFD 분해 후 결합 부호 제거) + 소문자화 + 공백/아포스트로피 제거 |
| `isHanziInput(s)` | 입력값이 한자인지 판별 (병음/한자 둘 다 정답 인정용) |
| `checkAnswer(word, input)` | 입력이 한자면 `word.hanzi`와 완전일치, 아니면 `normalize()`한 병음끼리 비교 |
| `recordAnswer(word, outcome)` | 정답이면 wrong 카운트 감소, 오답이면 증가, 패스면 passed+wrong 증가. 복습 모드 우선순위에 반영됨 |

### 5-3. 화면 (screen state machine)
`state.mode` (탭) + `state.screen` (하위 화면)에 따라 `render()`가 아래 함수 중 하나를 호출합니다.

| mode | screen | 렌더 함수 | 설명 |
|---|---|---|---|
| `quiz` | `setup` | `renderSetup()` | 범위/모드 선택 화면 |
| `quiz` | `quiz` | `renderQuiz()` | 병음/한자 입력 시험. `Enter`=확인/다음, `Tab`=패스 |
| `quiz` | `result` | `renderResult()` | 시험 결과 + 오답/패스 목록 |
| `sentence` | — | `renderSentence()` | 문장 만들기 + 규칙 기반 채점 (3-1 참고) |
| `notes` | — | `renderNotes()` | 전체 기간 오답노트/패스노트 (탭 전환) |
| `manage` | — | `renderManage()` | 단어장 검색/추가/수정/삭제 + 백업 내보내기·가져오기 |

## 8. 코드 컨벤션

- 화면 렌더는 항상 `app.innerHTML = \`...\`` 통짜 재생성 방식 (검색 필터처럼 매 렌더가 비싼 경우만 예외 — `renderManage()`의 검색 input 리스너 참고)
- 저장이 필요한 변경은 항상 `persist*()` 함수를 통해서만 (디바운스 300ms)
- 단어 객체 스키마: `{ id, category, group, hanzi, pinyin, meaning, note, starred }` (`category`와 `group`은 항상 같은 값)
- 카테고리는 항상 `GROUP_ORDER` 9개 중 하나로 고정 (자유 텍스트 카테고리 없음)
- 사용자 입력이 섞이는 모든 렌더 지점은 `h()` 함수로 HTML 이스케이프
