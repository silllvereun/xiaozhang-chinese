# 작업서 — 샤오장의 중국어학습

VSCode에서 바이브코딩으로 이어서 작업할 수 있도록 정리한 인수인계 문서입니다.
이 문서 + `index.html` 하나만 있으면 어떤 AI 코딩 툴(Claude Code, Cursor, Copilot 등)에게 그대로 붙여넣고 작업을 이어갈 수 있어요.

> 이 문서는 2026-07-03에 처음부터 새로 만든 버전을 기준으로 작성되었습니다. Claude 아티팩트 환경(`window.storage`, 인증 없는 Claude API 직접 호출)에 의존하던 이전 설계는 배제하고, 처음부터 독립 배포(GitHub Pages 등) 가능한 형태로 구현했습니다. 2026-07-04에 Supabase 연동(공유 단어장 + 로그인 기반 개인 학습 기록 동기화)을 추가했습니다.

---

## 1. 프로젝트 개요

- **이름**: 샤오장의 중국어학습
- **형태**: 빌드 도구 없는 단일 HTML 파일 (Vanilla JS, 프레임워크 없음) + Supabase 백엔드
- **목적**: 개인용 중국어 단어 암기 + 문장 작문 연습 웹앱
- **데이터**: 391개 단어(HSK1 111 · HSK2 93 · HSKK초급 25 · 회화 30 · 업무 30 · 금융 23 · 거래소 23 · IT/개발 30 · 암호화폐 26). Supabase `words` 테이블에 저장되어 있고, 위키백과처럼 로그인 없이 누구나 조회·추가할 수 있습니다.
- **파일**: `index.html` 1개 (HTML+CSS+JS 전부 인라인, Supabase JS SDK만 CDN으로 로드)

## 2. 기술 스택 & 선택 이유

| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | 없음 (Vanilla JS) | 이 규모(1인 사용자, 정적 데이터)에 React 등은 과설계. 빌드 스텝 없이 파일 하나로 배포 가능 |
| 단어장 저장소 | **Supabase (Postgres) `words` 테이블** | 위키백과 스타일 공유 단어장. RLS로 SELECT/INSERT는 누구나(anon), UPDATE/DELETE는 로그인한 사용자만 허용 |
| 개인 학습 기록 | 로그인 전: `localStorage` / 로그인 후: Supabase `word_progress` 테이블과 동기화 | 로그인 없이도 그 기기에서 계속 문제를 풀 수 있고, 이메일 매직링크로 로그인하면 그 순간부터 기록이 계정에 귀속되어 다른 기기에서도 이어짐 |
| 인증 | Supabase Auth, 이메일 매직링크 (비밀번호 없음) | 가장 간단한 방식. `supabase.auth.signInWithOtp()` |
| 배포 | GitHub Pages | 무료, 정적 파일이라 서버 불필요. `https://silllvereun.github.io/xiaozhang-chinese/`에 배포 완료 |
| 문장 채점 | **규칙 기반(항상) + 무료 AI 첨삭(선택, 프록시 경유)** | 규칙 기반(`gradeSentence()`)이 항상 먼저 동작: 목표 단어 포함 여부, 문장 길이(한자 4자 이상), 문장부호, 한글 미포함 4가지 검사. 형식 통과 시에만, `AI_PROXY_URL`이 설정되어 있으면 Cloudflare Worker(`worker.js`, Workers AI 무료 티어)로 문법 첨삭을 추가 호출(`gradeSentenceWithAI()`). 미설정이거나 실패하면 조용히 규칙 기반 결과로 폴백 — 서버 없이도 항상 동작함 |
| TTS(발음) | 브라우저 내장 `SpeechSynthesis`, 명시적으로 `zh` 음성을 찾아서 지정 | 별도 API 키 불필요, 무료. 다만 기기에 중국어 음성 팩이 설치되어 있어야 함 (3-3 참고) |

## 3. 알아둬야 할 제약사항

### 3-1. 문장 채점 — 규칙 기반 + 선택적 AI 첨삭 (구현 완료, 배포만 하면 됨)
`gradeSentence(word, text)`는 4가지 형식적 조건만 확인합니다: (1) 목표 한자 단어 포함, (2) 한자 4자 이상, (3) 문장부호(。！？!?)로 끝맺음, (4) 한글 미포함. 어순이나 조사, 문법 오류는 잡아내지 못합니다. 형식이 틀리면 AI를 호출하지 않고 바로 이 결과만 보여줍니다(무료 할당량 절약).

형식을 통과하면, `index.html`의 `AI_PROXY_URL` 상수가 설정되어 있을 때만 `gradeSentenceWithAI()`가 그 URL로 `fetch`해서 무료 AI(Cloudflare Workers AI, Qwen 14B) 문법 첨삭을 추가로 보여줍니다. 코드와 배포용 Worker(`worker.js`)는 이미 준비되어 있고, 배포 절차는 [DEPLOY_AI_PROXY.md](DEPLOY_AI_PROXY.md)에 정리되어 있습니다(약 5분). **남은 일은 그 가이드대로 Cloudflare Worker를 배포하고 발급된 URL을 `AI_PROXY_URL`에 채워 넣는 것뿐**입니다. `AI_PROXY_URL`이 비어 있으면 지금처럼 규칙 기반 채점만 동작합니다.

무료 Qwen 14B는 개인 학습 보조로는 충분하지만 Claude 같은 최상위 모델보다 첨삭 정확도가 낮습니다(특히 어색한 문장을 통과시키는 경향). 품질을 높이고 싶으면 `worker.js`의 모델명을 다른 Workers AI 모델로 바꾸거나, Workers AI 호출 부분을 Anthropic API 호출로 교체하면 됩니다.

### 3-2. 단어장은 공유, 학습 기록(즐겨찾기 포함)은 개인
`words` 테이블은 로그인 여부와 상관없이 전역으로 공유됩니다 — 새 단어 추가는 누구나 가능하고, 기존 단어 수정/삭제는 로그인한 사용자만 가능하도록 RLS로 제한했습니다(완전 무방비 훼손 방지 목적, 필요하면 더 열거나 닫을 수 있음).

반면 오답/정답/패스 횟수와 즐겨찾기(★)는 **개인 데이터**라서 `words`가 아니라 `word_progress` 테이블에 `user_id` 기준으로 저장됩니다. 로그인하지 않은 상태에서는 이 데이터가 그 브라우저의 `localStorage`에만 있고, 로그인하는 순간 서버와 병합·동기화됩니다 (`syncProgressAfterLogin()` 참고 — 서버에 있는 값이 우선 적용되고, 아직 서버에 없는 로컬 기록은 그대로 업로드됩니다).

### 3-3. TTS가 중국어로 안 들리면
`speak()`는 설치된 음성 중 `zh`로 시작하는 언어의 음성을 찾아서 명시적으로 지정합니다. 만약 기기에 중국어 음성이 설치되어 있지 않으면 시스템 기본 음성(예: 한국어)으로 대체되며, 이때 알림으로 안내 문구가 한 번 뜹니다. 안드로이드는 설정 → 언어 및 입력 → TTS 출력에서, iOS는 설정 → 손쉬운 사용 → 말하기 콘텐츠에서 중국어 음성을 추가해야 합니다.

## 5. Supabase 스키마

```sql
create table words (
  id text primary key,
  category text not null,
  hanzi text not null,
  pinyin text not null,
  meaning text not null,
  note text default '',
  created_at timestamptz default now()
);
alter table words enable row level security;
create policy "words are viewable by everyone" on words for select using (true);
create policy "anyone can add words" on words for insert with check (true);
create policy "logged in users can edit words" on words for update using (auth.role() = 'authenticated');
create policy "logged in users can delete words" on words for delete using (auth.role() = 'authenticated');

create table word_progress (
  user_id uuid references auth.users(id) on delete cascade not null,
  word_id text references words(id) on delete cascade not null,
  wrong int default 0,
  correct int default 0,
  seen int default 0,
  passed int default 0,
  last_seen timestamptz,
  starred boolean default false,
  primary key (user_id, word_id)
);
alter table word_progress enable row level security;
create policy "users manage their own progress" on word_progress
  for all using (auth.uid() = user_id) with check (auth.uid() = user_id);
```

- 프로젝트: `csaracwmfpdyxwyqiois` (Supabase URL/anon key는 `index.html`의 `SUPABASE_URL`/`SUPABASE_ANON_KEY` 상수 참고)
- Authentication → URL Configuration → Site URL이 `https://silllvereun.github.io/xiaozhang-chinese/`로 설정되어 있어야 매직링크가 올바르게 리다이렉트됩니다.
- anon key는 클라이언트 코드에 그대로 노출되지만, 실제 접근 제어는 위 RLS 정책이 담당하므로(키 자체가 아니라 정책이 방어선) 안전합니다.

## 6. 코드 아키텍처 요약

상태 관리는 전역 `state` 객체 하나 + `render()` 함수가 매번 `app.innerHTML`을 통짜로 다시 그리는 단순한 패턴입니다 (가상 DOM 없음). 검색 필터(`단어장 관리`)만 예외적으로 `renderManage()` 내부에서 DOM을 직접 필터링해 매 타이핑마다 전체 재렌더를 피합니다.

### 6-1. 데이터 계층
| 함수/변수 | 역할 |
|---|---|
| `RAW_WORDS` / `buildInitialWords()` / `INITIAL_WORDS` | 초기 시드 데이터(391개) 정의. Supabase 연결 실패 시 오프라인 폴백으로만 쓰임 |
| `supabase` | `createClient(SUPABASE_URL, SUPABASE_ANON_KEY)`로 만든 클라이언트 |
| `WORDS` (let, 배열) | 실제 런타임에 쓰이는 단어 목록. `loadWordList()`가 Supabase `words`에서 불러오고, 실패하면 `localStorage`의 `wordListCache` → 그것도 없으면 `INITIAL_WORDS` 순서로 폴백 |
| `wordStats` (let, 객체) | `{ [wordId]: {wrong, correct, seen, passed, lastSeen, starred} }` — 개인 학습 통계 + 즐겨찾기. 로그인 전엔 `localStorage`(`wordStats`)에만, 로그인 후엔 `word_progress`에도 동기화 |
| `currentUser` | 로그인한 Supabase 사용자 객체 또는 `null` |
| `dbGet/dbSet/dbDelete(key)` | `localStorage` 얇은 래퍼 (캐시/오프라인 폴백용) |
| `loadWordList()` | Supabase에서 단어 목록 조회, 실패 시 캐시/시드로 폴백 |
| `loadStats()` | `localStorage`에서 `wordStats` 복원 |
| `addWord / updateWord / deleteWord` | `words` 테이블에 직접 insert/update/delete. 실패 시(로그인 필요 등) alert로 사유 표시 |
| `toggleStar(id)` | `wordStats[id].starred` 토글 후 `persistStats()` (더 이상 `words` 테이블과 무관) |
| `isStarred(id)` | 렌더링에서 즐겨찾기 여부 조회용 헬퍼 |
| `initAuth() / sendMagicLink() / signOut()` | Supabase Auth 세션 관리 |
| `syncProgressAfterLogin()` | 로그인 시 서버의 `word_progress`를 로컬 `wordStats`에 병합(서버 값 우선) 후 로컬에만 있던 기록을 다시 업로드 |
| `pushStatsToServer()` | `persistStats()`가 로그인 상태면 함께 호출 — `wordStats` 전체를 `word_progress`에 upsert |

### 6-2. 출제 로직
| 함수 | 역할 |
|---|---|
| `pool()` | 현재 필터(`selectedGroups`, `starredOnly`→`isStarred()`, `reviewMode`) 기준으로 후보 단어 반환 |
| `weightedSample(arr, n)` | 오답 횟수(`wrongCountOf`) + 안 본 지 오래된 정도(`daysSinceSeen`)를 가중치로 반영한 비복원 랜덤 샘플링 |
| `normalize(s)` / `stripTones(s)` | 병음 정답 비교용 — 성조 기호(NFD 분해 후 결합 부호 제거) + 소문자화 + 공백/아포스트로피 제거 |
| `isHanziInput(s)` | 입력값이 한자인지 판별 (병음/한자 둘 다 정답 인정용) |
| `checkAnswer(word, input)` | 입력이 한자면 `word.hanzi`와 완전일치, 아니면 `normalize()`한 병음끼리 비교 |
| `recordAnswer(word, outcome)` | 정답이면 wrong 카운트 감소, 오답이면 증가, 패스면 passed+wrong 증가. 복습 모드 우선순위에 반영됨 |

### 6-3. 화면 (screen state machine)
`state.mode` (탭) + `state.screen` (하위 화면)에 따라 `render()`가 아래 함수 중 하나를 호출합니다. 로그인 바(`renderAuthBar()`/`bindAuthBar()`)는 탭과 무관하게 항상 최상단에 렌더링됩니다.

| mode | screen | 렌더 함수 | 설명 |
|---|---|---|---|
| `quiz` | `setup` | `renderSetup()` | 범위/모드 선택 화면 |
| `quiz` | `quiz` | `renderQuiz()` | 병음/한자 입력 시험. `Enter`=확인/다음, `Tab`=패스 |
| `quiz` | `result` | `renderResult()` | 시험 결과 + 오답/패스 목록 |
| `sentence` | — | `renderSentence()` | 문장 만들기 + 규칙 기반 채점 (3-1 참고) |
| `notes` | — | `renderNotes()` | 전체 기간 오답노트/패스노트 (탭 전환) |
| `manage` | — | `renderManage()` | 단어장 검색/추가/수정/삭제 + 백업 내보내기·가져오기(Supabase upsert) |

4. **[선택] 오프라인 대응 강화** — 현재는 Supabase 요청 실패 시 캐시/시드로 조용히 폴백만 함. 사용자에게 "오프라인 모드" 표시를 추가하면 더 명확해짐
5. **[선택] 단어 훼손 모니터링** — `words`는 로그인 없이 누구나 추가 가능(위키 스타일)하므로, 배포 URL이 알려질 경우 스팸성 단어가 추가될 수 있음. 필요시 INSERT도 로그인 필수로 좁히거나, Supabase 대시보드에서 주기적으로 검수

## 9. 코드 컨벤션

- 화면 렌더는 항상 `app.innerHTML = \`...\`` 통짜 재생성 방식 (검색 필터처럼 매 렌더가 비싼 경우만 예외 — `renderManage()`의 검색 input 리스너 참고)
- 개인 학습 기록 변경은 항상 `persistStats()`를 통해서만 (디바운스 300ms, 로그인 시 자동으로 서버에도 upsert)
- 단어장(`words`) 변경은 `addWord/updateWord/deleteWord`를 통해서만 — Supabase 호출 성공 후에만 로컬 `WORDS` 배열도 갱신
- 단어 객체 스키마: `{ id, category, group, hanzi, pinyin, meaning, note }` (`starred`는 더 이상 여기 없음 — `wordStats[id].starred` 참고. `category`와 `group`은 항상 같은 값)
- 카테고리는 항상 `GROUP_ORDER` 9개 중 하나로 고정 (자유 텍스트 카테고리 없음)
- 사용자 입력이 섞이는 모든 렌더 지점은 `h()` 함수로 HTML 이스케이프
