# 작업서 — 한자 관계망 기능 (漢字 그래프)

`작업서.md`의 확장 문서입니다. 단어를 한자 단위로 분해해서, 한자 ↔ 단어 ↔ 문장이 서로 연결되는 관계망을 만드는 기능의 백엔드 설계와 UI/UX 설계를 담습니다.

> 📌 예시 정정: 출금은 `提现 (tíxiàn)`이에요 — 提(끌 제) + 现(나타날 현). `体现 (tǐxiàn)`은 "구현하다/드러내다"라는 다른 단어인데, 발음이 비슷해서 헷갈리기 쉬워요. 오히려 이런 혼동이야말로 이 기능이 잡아줄 수 있는 부분이라 좋은 예시입니다.

---

## 1. 기능 정의 (한 문장씩)

1. 모든 단어는 한자 단위로 분해되어 저장된다 — `提现` → `提` + `现`
2. 각 한자는 훈음(한국식: "나타날 현"), 병음, 뜻, 어원 설명을 가진다
3. 앱 안 어디서든 한자를 탭하면 → 그 한자의 상세(훈음·뜻·어원)와, **그 한자가 들어간 다른 단어 목록**이 뜬다 — `现` → `提现`(출금), `现货`(현물), `发现`(발견하다), `出现`…
4. 문장 만들기에서 작성한 문장은 사용된 단어와 연결 저장된다 → 단어 상세에서 "이 단어를 쓴 내 문장들"을 볼 수 있다
5. 결과적으로 3계층 그래프가 생긴다: **한자(字) ↔ 단어(词) ↔ 문장(句)**

## 2. 데이터 모델 (Supabase / Postgres)

기존 `작업서.md` 7-1의 `words`, `word_stats` 테이블에 3개 테이블을 추가합니다.

```sql
-- ① 한자 사전 (한 글자 = 한 행)
create table characters (
  hanzi        text primary key,          -- '现' (글자 자체가 PK, 자연키)
  pinyin       text not null,             -- 'xiàn'
  korean_name  text,                      -- '나타날 현' (훈음)
  meaning      text,                      -- '나타나다, 현재'
  etymology    text,                      -- 어원/구성 설명 (예: '王(玉)+见 — 옥의 광채가 눈에 보인다')
  created_at   timestamptz default now()
);

-- ② 단어-한자 연결 (다대다 + 위치)
create table word_characters (
  word_id   text references words(id) on delete cascade,
  hanzi     text references characters(hanzi) on delete cascade,
  position  int not null,                 -- 단어 내 글자 순서 (0부터)
  primary key (word_id, position)
);

-- ③ 내가 만든 문장
create table sentences (
  id          text primary key,           -- 클라이언트 생성 id
  content     text not null,              -- '我想提现一千元。'
  translation text,                       -- 한국어 뜻 (선택)
  feedback    text,                       -- 채점 피드백 (규칙 기반 or AI)
  passed      boolean,
  created_at  timestamptz default now()
);

-- ④ 문장-단어 연결 (어떤 단어 연습으로 만든 문장인지 + 문장에 실제 포함된 단어들)
create table sentence_words (
  sentence_id text references sentences(id) on delete cascade,
  word_id     text references words(id) on delete cascade,
  is_target   boolean default false,      -- true = 이 단어를 연습하려고 만든 문장
  primary key (sentence_id, word_id)
);
```

**설계 포인트:**
- `characters.hanzi`를 자연키(PK)로 씀 — 글자 자체가 유일하고 불변이라 인조키가 불필요
- `word_characters.position` 덕분에 `提现`과 `现提`(가상의 예) 구분 가능, 단어 내 글자 순서대로 렌더링 가능
- `sentence_words.is_target`으로 "이 단어 연습용으로 쓴 문장"과 "문장에 우연히 포함된 다른 단어"를 구분 → 단어 상세에서 둘 다 보여주되 구분 표시 가능
- 한자→단어 역조회는 `word_characters`를 `hanzi`로 인덱스 조회하면 끝 (별도 테이블 불필요):
  ```sql
  create index idx_wc_hanzi on word_characters(hanzi);
  create index idx_sw_word on sentence_words(word_id);
  ```

## 3. 데이터 구축 전략 (제일 중요한 현실 문제)

391개 단어 × 평균 2자 = 약 500~600개 한자 행이 필요한데, 훈음·어원을 손으로 다 입력하는 건 비현실적입니다.

**권장 절차 (1회성 시드 작업):**
1. 기존 단어 목록에서 유니크 한자 추출 (JS 한 줄: `[...new Set(WORDS.flatMap(w => [...w.hanzi].filter(c => /[一-鿿]/.test(c))))]`)
2. 추출된 한자 목록을 Claude(또는 Claude Code)에 한 번에 넘겨서 `{hanzi, pinyin, korean_name, meaning, etymology}` JSON 일괄 생성 — 한자 훈음·어원은 LLM이 잘하는 영역이지만 **오류가 섞일 수 있으니 "AI 생성 초안" 표시를 달고, 앱에서 한자 상세를 볼 때 수정 버튼을 제공** (사용하면서 자연스럽게 검수)
3. `word_characters`는 코드로 자동 생성 (단어 문자열 분해 → position 부여). 단, `接口 / API`처럼 한자가 아닌 문자가 섞인 단어는 한자만 추출
4. 새 단어를 추가할 때: 자동으로 글자 분해 → `characters`에 없는 글자면 "이 한자의 정보가 아직 없어요" 상태로 두고, 상세 화면에서 채워넣거나 AI 초안 버튼으로 생성

**다음절 단어의 함정 (주의):**
- `了`(le/liǎo), `长`(cháng/zhǎng), `行`(xíng/háng)처럼 다음자(多音字)는 단어마다 발음이 다름 → `characters.pinyin`은 대표 발음만 두고, 필요하면 `word_characters`에 `pinyin_override` 컬럼을 추가하는 게 확장 여지. 초기 버전에서는 대표 발음만으로 충분.

## 4. 클라이언트 데이터 계층 변경

기존 `dbGet/dbSet/dbDelete` 추상화 위에 조회 함수 4개만 추가하면 됩니다:

| 함수 | 반환 | 쓰이는 곳 |
|---|---|---|
| `getCharacter(hanzi)` | 한자 상세 1건 | 한자 시트 |
| `getWordsByCharacter(hanzi)` | 그 한자가 포함된 단어 목록 | 한자 시트의 "관련 단어" |
| `getSentencesByWord(wordId)` | 그 단어가 쓰인 내 문장 목록 | 단어 상세의 "내 문장" |
| `saveSentence(sentence, wordIds, targetWordId)` | — | 문장 만들기 채점 통과 시 자동 호출 |

localStorage 단계(Supabase 전환 전)에서도 동일하게 동작 가능: `characters`, `sentences`, `sentence_words`를 각각 JSON 키로 저장하고 클라이언트에서 필터링하면 됨. **즉 이 기능은 Supabase 전환을 기다릴 필요 없이 먼저 구현 가능.**

## 5. UI/UX 설계 — 원칙: "새 화면을 만들지 말고, 이미 있는 한자를 전부 입구로 만든다"

### 5-1. 핵심 인터랙션 하나로 통일: 한자 탭 → 하단 시트(bottom sheet)

새로운 메뉴/탭을 추가하지 않습니다. 대신 **앱 안에 이미 표시되는 모든 한자를 탭 가능하게** 만들고, 탭하면 화면 아래에서 시트가 올라옵니다. 배우다가 궁금한 순간 그 자리에서 바로 파고들 수 있는 구조 — 이게 가장 심플하면서 학습 흐름을 안 끊는 방식입니다.

```
┌─────────────────────────┐
│  (기존 화면 그대로, 위쪽 어둡게)   │
├─────────────────────────┤
│  现  xiàn        🔊  ✎  │   ← 큰 글자 + 발음 + 수정
│  나타날 현                 │   ← 훈음
│  나타나다, 현재             │   ← 뜻
│  ─────────────────────  │
│  어원: 王(玉)+见. 옥의 광채가  │   ← 어원 (없으면 "AI 초안 생성" 버튼)
│  눈에 드러나 보인다는 뜻       │
│  ─────────────────────  │
│  이 한자가 들어간 단어 (4)     │
│  提现 tíxiàn   출금        │   ← 탭하면 단어 상세로
│  现货 xiànhuò  현물        │
│  发现 fāxiàn   발견하다      │
│  出现 chūxiàn  나타나다      │
└─────────────────────────┘
```

- 시트는 화면 절반 높이, 위로 드래그하면 전체 화면, 아래로 스와이프/바깥 탭하면 닫힘
- 시트 안의 단어를 탭하면 → **단어 시트**로 내용 교체 (시트 안에서 네비게이션, 뒤로가기 지원)
- 시트 안의 단어 속 한자를 또 탭하면 → 다시 한자 시트 (무한 탐색 가능 = 관계망을 "돌아다니는" 경험)

### 5-2. 단어 시트 (한자 시트와 쌍)

```
┌─────────────────────────┐
│  提现  tíxiàn     🔊  ★ │
│  출금                    │
│  ─────────────────────  │
│  글자 구성                 │
│  [提 끌 제] + [现 나타날 현] │   ← 각각 탭 가능 (한자 시트로)
│  ─────────────────────  │
│  이 단어를 쓴 내 문장 (2)    │
│  · 我想提现一千元。 ✓통과   │
│  · 提现手续费太贵了。 ✓통과  │
│  ─────────────────────  │
│  오답 3회 · 최근 학습 2일 전  │   ← 학습 통계 연결
└─────────────────────────┘
```

### 5-3. 한자 탭 가능 지점 (전부 기존 화면)

| 화면 | 한자가 나오는 곳 | 동작 |
|---|---|---|
| 시험 피드백 | 정답 공개된 한자 | 글자별로 탭 → 한자 시트 |
| 문장 만들기 | 단어 카드의 큰 한자 | 글자별로 탭 |
| 오답노트/패스노트 | 목록의 한자 | 글자별로 탭 |
| 단어장 관리 | 목록의 한자 | 글자별로 탭 |
| 시트 내부 | 관련 단어, 글자 구성 | 시트 안에서 이동 |

**시각적 힌트**: 탭 가능한 한자는 아주 옅은 점선 밑줄 (기존 잉크·인장 테마 유지, 파란 링크색 금지 — 디자인 톤 깨짐)

### 5-4. 문장 만들기와의 자동 연결 (사용자가 아무것도 안 해도 됨)

1. 문장 채점 통과 시 → 문장 자동 저장 + 연습 대상 단어와 `is_target=true`로 연결
2. 저장 시 문장 텍스트를 스캔해서 **단어장에 있는 다른 단어**가 포함돼 있으면 `is_target=false`로 함께 연결 (최장 일치 우선)
3. 결과: "我想提现一千元" 저장 → `提现`(target), `想`, `元`, `一`이 전부 연결 → 나중에 `想`의 단어 시트에서도 이 문장이 보임

### 5-5. 만들지 않기로 한 것 (스코프 아웃)

- ❌ 그래프 시각화(노드-엣지 다이어그램) — 멋있지만 폰 화면에서 실용성 낮음. 시트 네비게이션이 같은 정보를 더 편하게 전달
- ❌ 별도 "한자 사전" 탭 — 입구를 늘리면 오히려 복잡해짐. 검색이 필요하면 기존 단어장 관리 검색에 한자 결과를 섞는 것으로 충분
- ❌ 한자 필기 연습 — 별개 기능, 나중에 논의

## 6. 구현 순서 (권장)

1. **P1 — 데이터**: 한자 추출 스크립트 + AI 초안 일괄 생성 + `characters`/`word_characters` 시드 (localStorage 키로 먼저)
2. **P1 — 한자 시트**: 시트 컴포넌트 + 시험 피드백 화면의 한자부터 탭 가능하게 (가장 자주 보는 곳)
3. **P2 — 단어 시트** + 시트 내부 네비게이션
4. **P2 — 문장 자동 저장·연결** (규칙 기반 채점 구현과 같은 타이밍에)
5. **P3 — 나머지 화면들의 한자 탭 확대** + 한자 정보 수정 UI
6. **P3 — Supabase 전환 시 위 테이블 스키마로 이관** (기존 작업서 TODO 1번과 병합)
